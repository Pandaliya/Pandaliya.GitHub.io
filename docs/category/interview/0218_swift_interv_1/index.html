    Swift面试刷题（一） #

    swift基础面试题

    01 swift枚举的关联值和原始值怎么区分？ #

    关联值：是指将枚举成员和其他类型的变量存储在一起 原始值：枚举成员可以使用相同类型的默认值预先绑定，这个默认值叫做原始值

    enum Date {
    // 设置枚举成员关联值
    case digit(year:Int, month:Int, day:Int)
    case string(String)
}

enum Grade: String {
    case perfect = "A"
    case great = "B"
    // ...
}

    02 什么是延时存储属性？ #

    使用lazy可以定义一个延时存储属性，在第一次使用时才进行初始化；

    使用lazy的注意点：

    1. lazy必须是var属性
    2. lazy线程不安全，多线程同时访问时，无法保证只初始化一次
    // 定义lazy属性
class PicView {
    lazy var image: Image = {
        return Image(data:data)
    }
}

    03 什么是运算符重载？ #

    类、结构体、枚举可以为现有的运算符提供自定义实现

    // 代码示例
struct Point {
    var x: Int
    var y: Int

    static func + (p: Point1, p2: Point) {
        return Point(x: p1.x+p2.x, y: p1.y+p2.y)
    }
}

    04 定义静态方法时static和class的区别？ #

    static定义的方法不可以被子类覆写（override）；class则可以

    05 swift中如何定义下标？ #

    在swift中可以使用subscript给任意类型（枚举，结构体，类）定义下标

    06 说明swift中的初始化器 #

    类/结构体/枚举都可以定义初始化器，结构体有默认的初始化器。类有两种初始化器，分别为：指定初始化器和便捷初始化器（convenience init），规则如下。

    1. 每个类至少有一个指定初始化器。
    2. 指定初始化器必须调用父类的指定初始化器；便捷初始化必须调用当前类的另一个初始化器，且最终必须调用一个指定初始化器。

    07 Swift和OC的初始化方法有什么不同？ #

    1. swift的初始化方法更加严格，初始化方法必须保证所有非optional的成员都完成初始化；
    2. swift提供convenience（便捷）初始化器

    08 swift有那些闭包？ #

    尾随闭包

    尾随闭包是一个书写在函数括号后面的闭包表达式，如果需要将一个很长的闭包表达式作为函数的最后一个参数传递给函数，则可以将这个闭包替换为尾随闭包；

    逃逸闭包和非逃逸闭包

    当一个闭包作为参数传递到一个函数中，但这个闭包在函数返回还会被执行，就叫作闭包从函数中逃逸。可以添加@escaping关键字指明这是一个逃逸闭包。

    // 定义一个数组用于存储闭包类型
var completionHandlers: [() -> Void] = []

//  在方法中将闭包当做实际参数,存储到外部变量中
func someFunctionWithEscapingClosure(completionHandler: @escaping () -> Void) {
    completionHandlers.append(completionHandler)
}

    自动闭包

    自动闭包不接受任何参数，延迟求值，只有被调用时才返回被包装在其中的表达式的值。可以使用@autoclosure声明一个自动闭包参数。

    func getName() -> String{
    print(#function)
    return "DKJone"
}

//@autoclosure
func goodMooning(morning:Bool ,who:@autoclosure() -> String){
    if morning {
        print("Good morning, (who())")
    }
}

    自动闭包有以下特性：

    1. @autoclosure 会自动将20封装成闭包 { 20 }
    2. @autoclosure 只支持 () -> T格式的参数
    3. @autoclosure 并非只支持最后一个参数
    4. 有无@autoclosure 会后构成函数重载

    函数重载：函数名相同，函数的参数个数或类型不同

    10 swift语法糖的本质（实现原理） #

    语法糖： 是指计算机语言中添加某项特殊的语法，这种语法对语言本身功能没影响。但更方便编程者使用。增强代码可读性，简化代码。

    swift语法糖包括：

    1. Selector
    2. if let
    3. guard
    4. ?(单问号) / ??(双问号：) /!(感叹号：强制解包)

    11 如何解决循环引用 #

    1. 转换为值类型
    2. delegate或闭包使用weak或unowned属性

    12 如何理解copy-on-write? #

    值类型（如struct）在复制时，复制对象和原对象在实际内存中指向同一个对象（内存地址）。只有修改复制对象时，才会在内存中创建一个新对象。

    为提升性能。Struct，String，Array，Dictionary，Set都使用了cooy-on-write技术

    13 swift为什么将String/Array/Dictionary设计为值类型 #

    1. 高效使用内存，值类型在栈上操作只是指针移动；引用类型在堆上
    2. 线程安全，值类型赋值是深拷贝操作，会从新创建一个新的对象，不同线程上的操作不相互影响

    14 什么属性观察？ #

    属性观察：是指在当前类对特定的属性进行检测。

    var title: String {
    willSet {
        print("willSet", newValue)

    }
    didSet {
        print("didSet", oldValue, title)
    }
}

    有如下特性：

    1. willSet会传递新值，默认叫做newValue
    2. didSet传递旧值
    3. 初始化器中设置属性值不会触发willSet和didSet

    15 swift自省和OC有什么区别？ #

    自省就是判断某一个对象是否属于某一个类型的操作；

    [obj isKindOfClass:[Object class]];
[obj isMemberOfClass:[Object class]];

    swift 使用is关键字

    if (obj is ClassA) {
    //
}

    16 Swift中的扩展和OC的分类有什么不同？ #

    1. 扩展不能添加存储属性（会报编译错误）；分类可以通过运行时关联对象的形式添加属性
    2. Swift的扩展可以作用于协议，结构体或枚举；分类只能作用于类
